R Markdown — Dynamic Documents for R

R Markdown is an authoring format that enables easy creation of dynamic documents, presentations, and reports from R. It combines the core syntax of markdown (an easy-to-write plain text format) with embedded R code chunks that are run so their output can be included in the final document. R Markdown documents are fully reproducible (they can be automatically regenerated whenever underlying R code or data changes).

With R Markdown you can author a single source file that can be used to generate output in many formats including HTML, PDF, and MS Word. R Markdown isn’t limited to just plain documents either, extensive support for HTML5 and PDF (Beamer) presentations is also included.

Under the hood R Markdown uses the knitr package for executing R code and dynamically generating output and pandoc to convert markdown into an HTML, PDF, or Word document. A quick tour of the basics of R Markdown is included below.

Using R Markdown

You can use R Markdown either through the the integrated support for R Markdown within RStudio or via the rmarkdown package. To get started:

  1. RStudio — Install the current RStudio Preview Release.

  2. Other envrionments — Install the rmarkdown package from GitHub.

Embedding equations

You can embed LaTeX or MathML equations in R Markdown files using the following syntax:

  • $equation$ for inline equations (note there must not be whitespace adjacent to the $ delimiters)
  • $$ equation $$ for display equations
  • <math ...> </math> for MathML equations.

Migrating to v2

The current implementation of R Markdown (v2) is an evolution of the original implementation, which was not based on pandoc but rather the markdown package. Moving to pandoc brings many new features to R Markdown however carries some minor incompatibiliites with the prevoius implementation, specifically:

  1. The syntax for superscript now requires a closing ^ (e.g. superscript^2^).
  2. The WordPress-style LaTeX equation syntax (e.g. $latex <equation>$) is no longer supported.
  3. Markdown is rendered even if it’s contained within HTML tags.

Note that if you are using RStudio you can force RStudio to render documents using R Markdown v1 by adding the following special comment to your source file:

<!-- rmarkdown v1 -->

For rendering R Markdown v1 documents outside of RStudio you can continue to use the markdown package.
